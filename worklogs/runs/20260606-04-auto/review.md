## Reasoning

LRA's central primitive is the intra-trajectory loop-signature aggregator `L[s,a] ∈ ℝ^k` — the empirical mean of per-channel cumulant deltas accumulated along closed within-episode paths (the agent literally returns to the same observation hash). The improvement operator is a single logit-suppression rule keyed on Pareto-dominance by the fixed zero vector: an action is suppressed only when its loop signature loses in at least one channel and gains in none — i.e., the detour was provably wasteful in vector terms. This is structurally distinct from every disqualifier family: it is not a Bellman backup, not a scalar-weighted log-prob update, not count-based exploration (which suppresses revisits regardless of vector content), not RND/curiosity (no novelty signal), and not HER (no virtual goals). The nearest prior attempts are BCE-v0 (which used returnability as a novelty/frontier bonus that ablated to a count bonus) and RSD (which compares parallel segments *across* trajectories). LRA differs from both in that the loop-entrance event is the dominance precondition, not an exploration bonus, and the comparison is intra-trajectory against the fixed reference zero rather than cross-trajectory against another segment's mean. The required candidate-shape slots are all filled with substantive content: one primitive, one operator, named side-information channels (transition geometry + vector diagnostics), a conditional monotonic improvement claim, explicit falsifiers with diagnostics, and a structural distinction paragraph.

The sprint-4 failure pattern ("any within-trajectory signal-geometry primitive collapses on terminal-only vector channels") applies here, and the hypothesis correctly identifies it in predicted failure mode #2 — but LRA is not simply a within-trajectory geometry primitive. The step-penalty channel does accumulate monotonically within every episode, so `L[s,a]` can be non-trivially negative even before reward fires. The risk is that on DST/RG the step-penalty channel is always negative for any looping action, making the Pareto-vs-zero test fire on every loop and collapsing the operator to "never revisit observations" — a count-suppression rebadge in those envs. The hypothesis flags this as a first-class falsifier and specifies an explicit diagnostic (per-channel sign histograms of `L[s,a]`); the family is declared dead if every vector env requires excluding the step-penalty channel, since that would be tuning, not a structural invariant. This is honest accounting, and the structural novelty survives the review gate — it is for the empirical run to adjudicate.

## Risks the Engineer should be aware of

- The most dangerous failure mode is the step-penalty-channel collapse on DST and RG: if the step channel is always negative across all `(s,a)` pairs, the Pareto-vs-zero test reduces to count suppression on those envs. The per-channel sign histogram diagnostic must be logged from episode 1 so the run can be terminated early if this is observed.
- Observation-hash collision rates are critical infrastructure: on DoorKey after key pickup the partial-observable state changes every step, so intra-trajectory loops may be near-zero. Log the collision rate diagnostic from the first rollout batch; if it stays below ~5% the operator is dead and the run should stop.
- The monotonic improvement claim is conditional on deterministic transitions and `n_min` samples per `(s,a)`. On stochastic envs (or with function-approximated hashing), `L[s,a]` may be noisy enough to suppress productive actions; the `n_min` threshold is the only guard and must be set conservatively rather than tuned to maximize panel score.
