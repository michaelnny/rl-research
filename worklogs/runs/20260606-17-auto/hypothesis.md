# 20260606-17-auto — ARP: Action-Reachable Pattern Lattice

## Research Gate

primitive: per-(state-cluster, action) empirical *support set* of binary channel-firing patterns observed in downstream suffixes — `S(s, a) ⊆ {0,1}^k` where each element is a binary vector marking which vector channels fired at least once in some completed suffix from `(s, a)`.
improvement_operator: at decision step in state-cluster `s`, nudge policy logit of action `a` upward iff there exists at least one pattern `b ∈ S(s, a)` that is a strict superset (coordinate-wise `≥` with at least one strict `>`) of *every* pattern in `S(s, a')` for some other action `a'`; the nudge magnitude is the count of `a'` that `a` strictly subsumes minus the count that strictly subsume `a`.
side_information: vector diagnostics, transition geometry
nearest_prior_or_disqualifier: GVFs/successor features (disqualifier family); also #14 Primal Behavior Flow Pivot, candidate VCC (vector-cumulant confluence).
falsifier: on the vector envs (Deep Sea Treasure, Resource Gathering) within the 120 s budget, log (a) the average `|S(s, a)|` across populated (s, a) cells, (b) the fraction of decision steps where the strict-superset operator fires on at least one action pair, and (c) the operator's accept/reject ratio. Falsifier triggers if (a) `|S(s,a)|` stays at 1 for ≥ 95% of cells (lattice degenerate), or (b) operator firing rate is < 1% across the budget (no discriminating structure between actions), or (c) panel score is below random on both vector envs.

## Mechanism

The primitive is a *combinatorial reachability set* — not a magnitude expectation, not a vector mean, not a Pareto front over cumulants. For each (state-cluster `s`, action `a`) we maintain `S(s, a)`: the deduplicated set of binary firing patterns `b ∈ {0,1}^k` where `b[m] = 1` iff channel `m` fired at least once in the suffix following the `(s, a)` transition. Each completed episode contributes one pattern to each visited `(s, a)` cell along its trajectory (the suffix from that step). The improvement operator is set-theoretic: action `a` is preferred over `a'` at `s` iff some pattern in `S(s, a)` strictly contains every pattern in `S(s, a')` under coordinate-wise `≥`. The logit nudge is `α · (n_a^{dom} − n_a^{sub})` where `n_a^{dom}` counts other actions whose entire support is strictly subsumed by some element of `S(s, a)`, and `n_a^{sub}` counts the reverse. There is no scalar collapse — the comparison is on binary support membership; there is no expectation — the comparison is on set existence; there is no temporal recursion — the lattice is updated by direct suffix observation, not bootstrap.

## Required candidate shape

1. **Experience object:** Completed episodes plus the per-step vector signal `info["vector"]`. For each step `t` of each episode we record the suffix's binary firing pattern `b_t = ⋁_{u ≥ t} 𝟙[v_u \neq 0]` (channel-wise OR over the suffix), along with the state-cluster id `s_t` (k-means or hash of the policy-net penultimate features at `o_t`) and the action `a_t`. No reward shaping, no goal labels, no event lens; only the binary "did this channel fire downstream" projection of the vector signal we already receive.
2. **Core primitive:** `S(s, a) ⊆ {0, 1}^k` — the empirical set of distinct downstream binary firing patterns. Stored as a hash-set per cell. The cardinality `|S(s, a)|` is bounded by `2^k` (`k ≤ 4` on our panel) so storage is constant per cell; total memory is `O(N_clusters · |A| · 2^k)`.
3. **Improvement operator:** At decision step in cluster `s`, for each action pair `(a, a')` compute `dom(a, a') = ⋁_{b ∈ S(s,a)} ⋀_{b' ∈ S(s,a')} (b ≥ b' ∧ b ≠ b')` — i.e., `S(s,a)` contains some element that strictly supersets every element of `S(s,a')`. Define `n_a^{dom} = |{a' : dom(a, a')}|` and `n_a^{sub} = |{a' : dom(a', a)}|`. Apply additive logit nudge `α (n_a^{dom} − n_a^{sub})`. Standard cross-entropy loss on this nudged target during a small policy-update phase; no critic, no return weighting, no Bellman.
4. **Execution rule:** At rollout time, sample actions from `softmax(logits + α (n_a^{dom} − n_a^{sub}))` where logits are produced by a policy network. Cluster id `s` is computed online from the network's penultimate features. Cells with no observations contribute zero nudge (uniform prior).
5. **Vector feedback rule:** Channels enter the primitive only as binary firing indicators (`v_m ≠ 0`). The improvement operator is set-theoretic strict-superset on `{0,1}^k`, never a linear combination, never a weighted sum. If the panel adds a 6th or 12th channel the lattice grows but the operator is unchanged: the partial order on `{0,1}^k` extends naturally.
6. **Rollout-cost discipline:** Each environment step contributes one pattern observation to `|τ|` cells (the suffix-OR is computed once per episode at episode end and stamped onto every visited cell). No counterfactual rollouts. No verifier calls. No simulator branching. One environment step ↔ one experience-object record. Updates are batched at episode-end; deployment cost is identical to a single-network policy with one extra clustering forward pass.
7. **Nearest-neighbor novelty audit:** Closest disqualifier is **GVFs/successor features** (#14-adjacent Primal Behavior Flow). Successor features estimate `E[Σ γ^t φ(s_t)]` — a *magnitude expectation* over a feature space — and extract policy by linear scalarization `wᵀ ψ(s, a)`. ARP estimates `support({⋁ 𝟙[v_u ≠ 0]})` — a *combinatorial set* over channel-firing patterns — and extracts policy by *strict-superset on the lattice `({0,1}^k, ≤)`*. The rebadge test: rename ARP's "binary firing pattern" to "feature expectation" and the operator's "strict-superset" must become "linear scalarization" — but it doesn't, because superset is not weight-based and admits no `wᵀ ·` representation. Closest candidate is **VCC** (vector-cumulant confluence): VCC indexes by quantized cumulant `c_t ∈ R^k` and tests Pareto dominance on `c_T − c_t ∈ R^k`. ARP indexes by state-cluster (which fills regardless of cumulant magnitude) and tests strict-superset on `{0,1}^k` (binary, not magnitude). Where VCC needs cumulant buckets to fill before the dominance margin fires, ARP needs only one episode that visits state `s` and reaches *any* terminal channel firing.
8. **Predicted failure modes:**
   - **Lattice degeneracy:** if the only channels that fire are step-penalty (fires every step) and one terminal channel that fires only on success, then before the first success every `(s, a)` cell contains exactly the singleton pattern `{step-penalty}` and the operator is silent. This is the bootstrap wall in its sharpest form. Falsifier (a) above is the diagnostic.
   - **Channel-correlation collapse:** if two channels always fire together in every suffix (e.g., gem and gold always co-occur on Resource Gathering), the lattice collapses from `2^k` patterns to fewer; the operator keeps working but loses discriminating power.
   - **Cluster collisions / non-Markov clusters:** if state-cluster `s` aggregates genuinely-different states with different reachability sets, `S(s, a)` becomes a union of distinct sets and the strict-superset test mis-fires (the agent prefers actions that "look good in some merged context" but not in this state). Mitigated by using policy-net penultimate features as cluster basis.
   - **Stochastic transition wash-out:** if from `(s, a)` the transition is highly stochastic, every action's reachable set becomes the same large set, and `dom` is universally false. This is the same Resource Gathering risk that killed CWAI; here it's gentler because membership is binary, not gradient-based.
   - **Action-redundancy:** if multiple actions are functionally equivalent at `s` (e.g., move-into-wall vs. no-op on MiniGrid), their supports are identical and the operator never breaks ties between them (intended behavior, but worth noting).
9. **Side-information channel:** {vector diagnostics, transition geometry}. Vector diagnostics provide the per-step channel-firing signal (`info["vector"]`); transition geometry provides the state-cluster basis through the policy-net penultimate features. No event traces, no demonstrations, no language description, no verifier feedback, no learned dynamics, no environment instrumentation beyond the standard `info["vector"]` already exposed by the panel.
10. **Monotonic improvement claim:** Under deterministic transitions and stable state-clustering, the operator monotonically increases the policy's expected *cardinality of reachable firing patterns* `E_{s ~ d^π}[|S(s, π(s))|]`. Proof sketch: the operator only nudges toward actions `a` whose `S(s, a)` strictly supersets every element of some other action's support; this never decreases reachability. Combined with the fact that strictly-larger reachable-pattern support implies that any subset-of-channels target (the goal in any vector-channel task) is reachable from at least one action whose `S` covers it, the policy's coverage of valuable channel-combinations is monotonically non-decreasing. Note: this does NOT claim monotone improvement of cumulative reward — it claims monotone improvement of the combinatorial reachability surrogate. The hypothesis is that on long-horizon sparse-reward tasks where reward is determined by reaching certain channel-firing combinations, this surrogate is the binding constraint.

## Why it is not GVFs / successor features

GVFs and successor features predict `ψ(s, a) = E[Σ γ^t φ(s_t) | s_0 = s, a_0 = a]` — a vector of channel-magnitude expectations under the policy — and extract behavior by `argmax_a wᵀ ψ(s, a)` for some weight vector `w`. ARP predicts `S(s, a) = {⋁_{u ≥ 0} 𝟙[v_u ≠ 0]}` — the set of distinct binary firing patterns — and extracts behavior by strict-superset on the lattice `({0,1}^k, ≤)`. Three structural distinctions: (1) the primitive is a SET, not a vector expectation, so there is no inner product `wᵀ ψ` that can recover it under any choice of `w`; (2) the operator is *strict-superset existence on an infinite set of indicator vectors*, which is non-linear and admits no scalarization recipe; (3) the primitive does not depend on suffix length, discount, or magnitude — only on whether each channel ever fires — so it is invariant to the per-channel scaling that makes scalarized successor features brittle.

## Why it scales beyond the substrate

When the action space becomes "generate a paragraph" or "call this tool with these arguments," vector channels still describe what the trajectory accomplished (validity passed, latency under threshold, user satisfaction recorded, safety violation avoided, correctness verified, etc.). The binary firing pattern is "which of these k events happened in this rollout." With `k = 6`-12 channels typical of LLM-tool-use deployments, `|S(s, a)| ≤ 2^k ≤ 4096` is a tractable storage object per (situation-cluster, action-prototype) cell. State-cluster generalization scales because we cluster on policy-net features, which are exactly what a transformer naturally produces. Action-cluster generalization is the hard part: at LLM scale, "actions" are paragraphs, so `S` must be indexed by an *action prototype* (a learned action-embedding cluster) rather than the raw paragraph; the strict-superset operator over patterns is unchanged. The primitive's bandwidth requirement (one suffix-OR per episode per (cluster, action) cell) scales with episodes, not with horizon length, so 20k-action episodes do not increase update cost — only the suffix-OR computation, which is `O(T · k)` per episode and trivially streamed. The fundamental bet is that *combinatorial reachability of channel-firing patterns* is the right abstraction layer for sparse-reward long-horizon discovery: agents should learn "from this situation, taking this action makes pattern `b` reachable that other actions don't" before they learn "this action's expected return is higher." That ordering is what makes the primitive bootstrappable when reward correlation has not yet emerged.
