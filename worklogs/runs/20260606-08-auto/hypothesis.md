# 20260606-08-auto — TCP (Temporal Channel Precedence)

## Research Gate

primitive: empirical lag-asymmetry tensor `Λ[j,k] = E_τ[ sign(t_first(j;τ) − t_first(k;τ)) ]` over ordered channel pairs (j,k), accumulated across the replay of full trajectories — a per-pair scalar in [−1,+1] that measures whether channel j tends to fire *after* channel k. From `Λ` we construct a precedence DAG `P` (edge `k→j` whenever `Λ[j,k]` exceeds a fixed asymmetry threshold), which is the operative primitive.

improvement_operator: at each decision step, given the trajectory prefix `τ_{:t}`, compute the *residual channel set* `R(τ_{:t}) = { j : channel j has not yet fired in τ_{:t} AND every predecessor of j in P has fired }` — the "next-eligible" channels in the precedence DAG. Nudge policy logits toward actions whose per-action-conditional next-step firing-vector empirical mean `μ[a, R(τ_{:t})]` (entries restricted to the residual channels, indexed by current observation cluster) is Pareto-non-dominated within `{μ[a', R(τ_{:t})] : a'}`. No scalarization across channels; the residual set itself shifts with prefix.

side_information: vector diagnostics (used to compute `Λ`), transition geometry (observation cluster used to condition `μ`).

nearest_prior_or_disqualifier: #15 FED / #18 CEC family (Pareto-front over per-action vector outcomes). Also adjacent to reward-machines disqualifier (channel precedence resembles automaton edges).

falsifier: if on Deep Sea Treasure or Resource Gathering the empirical `Λ` matrix is rank-deficient (all channels fire only at the same step, so all pairwise lag signs are zero) within the 120 s budget, the precedence DAG is empty and `R(·)` either equals all channels (no filtering) or is empty (no nudge fires). A panel score below random on the vector envs falsifies the family. Additionally, if the residual-channel-restricted Pareto comparison `μ[a, R]` collapses to a single channel for the majority of decision steps (because R is a singleton), the operator reduces to per-channel firing-frequency maximization — i.e. scalar reward weighting on one channel — and the candidate is a rebadge.

## Mechanism

The primitive is a per-channel-pair lag-asymmetry scalar `Λ[j,k] ∈ [−1, +1]` accumulated as the running sample-mean over trajectories of `sign( t_first(j;τ) − t_first(k;τ) )` (with `+1`/`−1`/`0` for "j after k"/"j before k"/"only one of them fires"). Asymmetry above a threshold τ_DAG induces a directed edge `k → j` in a precedence DAG `P` over the k vector channels. At every decision step, the agent identifies its trajectory's residual-channel set `R(τ_{:t})`: channels not yet fired whose DAG predecessors have all fired in the prefix. Conditioned on a coarse observation cluster (e.g. tile-coords for MiniGrid, learned k-means prototype id for pixels), the agent maintains `μ[c, a, m]`, the empirical mean of channel-m's firing indicator on the *next step* after taking action `a` in cluster `c`. The improvement operator nudges logits at each step by a Pareto-non-dominance count: action `a` gets `+α · #{a' : μ[c,a,R] ≻_Pareto μ[c,a',R]} − α · #{a' : μ[c,a',R] ≻_Pareto μ[c,a,R]}`, with the partial order taken only over the residual channel coordinates. Both `Λ` and `μ` update from full-trajectory passive replay — no counterfactual rollouts, no critic, no scalar collapse. The precedence DAG is the only object that compresses across trajectories; everything else is direct empirical conditional firing means.

## Required candidate shape

1. **Experience object:** Full trajectories (observations, actions, per-step vector signal `v_t ∈ ℝ^k`) collected by ordinary on-policy interaction. Per trajectory, two derived statistics are extracted: (a) per-channel first-firing time `t_first(m;τ)` (or `∞` if never), and (b) per-(cluster, action, next-channel-firing) increment counts `n[c,a,m] += 1[v_{t+1}[m] > 0]`.

2. **Core primitive:** The lag-asymmetry tensor `Λ ∈ [−1,+1]^{k×k}` (`Λ[j,k] = mean over trajectories of sign(t_first(j) − t_first(k))`) and its derived precedence DAG `P` (edge `k→j` iff `Λ[j,k] > τ_DAG`). `Λ` is a *cross-trajectory cross-channel temporal-asymmetry statistic*; it is not a Q, V, advantage, return-to-go, return distribution, successor feature, occupancy, or count.

3. **Improvement operator:** Logit nudge `Δ_logit(c,a) = α · ( #{a': μ[c,a,R] ≻_Pareto μ[c,a',R]} − #{a': μ[c,a',R] ≻_Pareto μ[c,a,R]} )` where `R = R(τ_{:t})` is the prefix-dependent residual channel set induced by `P`. Update is signed-Pareto-vote count, never a scalar reward weight, never bootstrap of a value target, never an elite-trajectory clone.

4. **Execution rule:** At decision step, sample action from softmax over (base policy logits + `Δ_logit(c(o_t), ·)`). The residual set `R` is recomputed from the live prefix every step; the nudge therefore changes both as `c(o_t)` changes spatially and as `R` shrinks temporally along the trajectory.

5. **Vector feedback rule:** Channels are never summed. `Λ` is computed channel-pair-wise. `μ[c,a,·]` is a vector. The Pareto comparison is restricted to the residual coordinate set `R`, which changes prefix-by-prefix. No `wᵀr` ever materializes — even implicitly — because the residual set is determined by the *temporal precedence* of channels (a structural property of `Λ`), not by their magnitudes.

6. **Rollout-cost discipline:** One environment step per stored experience tuple. Updates to `Λ` and `μ` happen at episode end and at every step respectively from the on-policy replay. No counterfactual branching, no verifier calls, no auxiliary rollouts, no simulator forks. Per accepted improvement: zero extra rollouts beyond the one trajectory that produced the update. Deployment cost: one cluster lookup + one residual-set computation + one Pareto-vote pass per action choice.

7. **Nearest-neighbor novelty audit:** Closest priors are #15 FED and #18 CEC, both of which compute per-action Pareto fronts over vector outcome multisets indexed by an observation hash. TCP is structurally distinct in two load-bearing ways: (a) the indexing object is *not* an observation hash but a prefix-induced residual-channel set, which is computed from the trajectory's own firing history through the lag-asymmetry DAG — this means TCP gates which channels it cares about based on temporal precedence, not which states it has visited; (b) the improvement signal does not require Pareto fronts to accumulate within a bucket before firing — it requires only that `Λ` becomes non-trivial, which happens as soon as any two channels fire at distinct times in any trajectory, even with no reward yet.

8. **Predicted failure modes:**
   - Fails on substrates where every vector channel fires only at the terminal step (`t_first` is identical for all channels in every trajectory): `Λ` is identically zero, DAG is empty, residual set is the full channel set, and the nudge reduces to per-(cluster, action) Pareto over all channels — i.e. FED with cluster instead of obs-hash. Specifically this is a real risk for Deep Sea Treasure where step penalty fires every step but treasure fires only at termination — `Λ` will pick up that step-penalty-precedes-treasure trivially, but won't have enough channels for the residual set to do useful filtering.
   - Fails when observation clustering is too coarse (μ is averaged across behaviorally distinct states) or too fine (μ is uninformative singletons). The cluster-granularity sensitivity is real.
   - Fails when channel precedence is environment-dependent and changes across episodes (e.g. in Resource Gathering an agent can pick up gold-then-gem or gem-then-gold; `Λ` for those two channels averages to zero and the DAG never connects them). This is the family-killer if channel precedence is intrinsically not a DAG on the substrate.
   - Fails on stochastic transitions where `μ[c,a,m]` has high variance — the Pareto-vote count flips between steps without learning.

9. **Side-information channel:** {vector diagnostics, transition geometry}. Vector diagnostics provide the per-channel firing-time and per-step firing indicators that drive `Λ` and `μ`. Transition geometry is used as the observation-cluster lens `c(o)` (tile-coords on MiniGrid; k-means prototype on Craftax). No demonstrations, no verifier, no learned dynamics, no environment instrumentation beyond what `info["vector"]` already exposes.

10. **Monotonic improvement claim:** Under the assumption that the channel-precedence DAG `P` is well-defined (no symmetric channel pairs above the asymmetry threshold) and stationary across episodes, AND that the residual-channel-restricted firing-rate vector `μ[c,a,R]` admits at least one Pareto-non-dominated action per cluster, the operator monotonically increases the expected residual-channel firing-rate (a vector quantity) at each cluster in the partial order sense. That is: for the ordered cone induced by `R(τ_{:t})`, the policy's induced firing-rate vector dominates the previous policy's in every coordinate of `R`. The improvement is in the vector dominance partial order, not in any scalar.

## Why it is not #15 FED / #18 CEC

FED and CEC both index per-action vector multisets by an *observation* hash and require enough samples per bucket to produce Pareto-front comparisons; they fail because long-horizon sparse envs starve the buckets. TCP's primitive is the cross-channel lag-asymmetry tensor `Λ`, whose entries fill from any trajectory in which two channels fire at distinct times — there is no observation-hash bucket to starve. The operator's *gate* is the prefix-induced residual channel set `R`, which is a function of the lag-asymmetry DAG and the live trajectory's firing history, not of state-revisit structure. Renaming `Λ` to a "transition matrix" or `R` to a "set of unfulfilled subgoals" does not collapse it to FED or CEC because the indexing object itself is structurally different (channel-pair temporal-asymmetry vs. state-action vector multiset).

## Why it is not reward-machines (disqualifier family)

Reward machines require a hand-specified or hand-discovered finite automaton over symbolic events; the automaton's transitions encode reward shaping logic and the operator is value backup over the joint (state × automaton) MDP. TCP's precedence DAG is *empirically discovered* from temporal asymmetry of channel firings — no human-specified events, no automaton transitions, no value backup over an extended state. The DAG only gates *which channels are eligible to drive Pareto comparisons at the current step*; it never modifies the reward signal nor extends the state space. There is no value function on a product MDP being learned anywhere.

## Why it scales beyond the substrate

At horizon 20k, the precedence DAG `P` is a fixed-size object (k×k where k is the channel count, ~6–12 in realistic vector-feedback settings) regardless of horizon — its statistics fill faster as trajectories grow longer because more channel-firing events are observed per trajectory. For LLM-tool-use settings the channels are naturally-given quantities (success, latency, cost, safety-flag, validity, user-preference); the lag-asymmetry primitive captures, e.g., "validity tends to fire before user-preference", which is a structural fact about the task that survives policy improvement. The residual-channel set `R(τ_{:t})` becomes a *prefix-conditional outcome target*: at any point in a long trajectory, the agent's vector "what is currently eligible to improve" is constructed from `P` and the prefix's firing history — it does not require trajectory-pair matching, observation-hash collision, or any accumulating bucket statistic that scales with horizon. The "generate a paragraph" action space is handled by clustering paragraph embeddings into `c`; the Pareto-vote operator over `R`-restricted firing means is well-defined regardless of action enumerability because per-action statistics `μ[c,a,·]` are aggregated for any action that gets sampled, not over all actions enumerated up-front.
