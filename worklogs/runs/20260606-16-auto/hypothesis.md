# 20260606-16-auto — CID (Channel-Imminence Differential)

## Research Gate

primitive: per-(observation, action, channel) one-step log-likelihood-ratio of the next-step firing probability of channel `m` between the *post-action* observation and the *current* observation, where firing probability is supplied by a small per-channel binary classifier `q_m(o)` trained by ordinary supervised learning on the replay buffer.
improvement_operator: Pareto-non-dominance vote on the k-dimensional vector `Δ[a,:] = (LR_1[a], …, LR_k[a])` followed by an additive logit nudge `α·(n_a − m_a)` where `n_a, m_a` are dominated/dominating counts in the coordinate-wise partial order on ℝ^k. No scalar collapse, no critic, no reward magnitude.
side_information: vector diagnostics (`info["vector"]` channels), learned dynamics (a one-step observation predictor used only to source the post-action observation when sampling a single rollout per decision is too expensive — but see Execution rule below; the canonical version simply uses the *actually observed* `o_{t+1}` to update CID statistics offline, no forward-model rollout at decision time).
nearest_prior_or_disqualifier: GVFs / successor features (disqualifier), CWAI (`worklogs/candidates/cwai-channel-wise-action-influence.md`), JFP (`worklogs/candidates/jfp-jacobian-firing-phase.md`), FFTV (`worklogs/candidates/fftv-first-firing-time-variance-lattice.md`).
falsifier: on a vector env where every precious channel is terminal-only AND the dense step-penalty channel is the only firing-active channel, `q_m(o)` for precious channels stays near zero everywhere, `LR_m ≈ 0` for those channels, and the Pareto vote degenerates to a single-channel scalar comparison on the step-penalty channel — equivalent to count-based exploration suppression. Specifically: if on Resource Gathering the gold/gem channel-fire fraction in the buffer stays below 0.01 after 90 s of training AND the step-penalty channel is the only one with non-zero `LR` row variance across actions, CID has degenerated and the run should be flagged as collapsed even if it scores above random on DST.

## Mechanism

CID's primitive is the per-channel one-step log-likelihood-ratio of imminent firing, indexed by `(o, a)`:

```
LR_m[o, a] = log q_m(o_{t+1}) − log q_m(o_t)
```

where `q_m: O → [0, 1]` is a small per-channel binary head trained by supervised log-likelihood on samples `(o_τ, I[v_{τ+1}[m] > 0])` drawn from the replay buffer. There are k such heads (one per channel), each shared across states and actions. To attribute the LR to a specific action choice at decision time, the operator evaluates `LR_m[o, a] = log q_m(ô_{t+1}(o, a)) − log q_m(o_t)` where `ô_{t+1}(o, a)` is the *empirical next-observation distribution from the buffer for (o, a)* — implemented as a single-step model `f_φ(o, a) → ô`, also trained on buffer transitions by L2 regression on observation deltas. (For pixel/large-obs envs, `f_φ` need only predict the small number of features `q_m` reads, not the full observation.) At each decision step the policy logits receive an additive nudge

```
Δlogit(a) = α · (n_a − m_a)
n_a = #{a' : LR[a, :] dominates LR[a', :] coordinate-wise, with strict in ≥ 1}
m_a = #{a' : LR[a', :] dominates LR[a, :] coordinate-wise, with strict in ≥ 1}
```

The signed Pareto-vote count is the only update channel; channels are never linearly combined, never weighted, never reduced to a scalar. No Bellman backup, no policy-gradient reward weighting, no critic.

## Required candidate shape

1. **Experience object:** Ordinary on-policy trajectories with per-step `info["vector"]` channels. The replay buffer stores tuples `(o_t, a_t, o_{t+1}, v_{t+1})`. No counterfactual rollouts, no verifier calls, no demonstrations, no hand-curated event boundaries.

2. **Core primitive:** The k-dimensional log-likelihood-ratio vector `LR[o, a, :]` where each component is the per-channel one-step imminence-shift `log q_m(ô_{t+1}(o, a)) − log q_m(o_t)`. `q_m` is a supervised binary classifier on observed firings; `ô_{t+1}` is a supervised one-step observation predictor. The primitive is bounded (`|LR_m| ≤ log(1/q_min)` after probability flooring), magnitude-stable under stochastic transitions (because `q_m` is a probability, not a Jacobian — stochasticity averages into `q_m` rather than shrinking it), and dimensionally exactly `k`.

3. **Improvement operator:** Pareto-non-dominance vote. For each candidate action `a` at decision time, compute `LR[a, :] ∈ ℝ^k`, then count Pareto dominations over alternative actions in the coordinate-wise partial order. The logit nudge is additive in domination margin. No scalar collapse at any point; channels are never combined.

4. **Execution rule:** Sample actions from softmax over `(base_logits + Δlogits)`. Base logits are produced by a small policy network trained jointly with `q_m` and `f_φ`. The base policy itself is updated by maximum-entropy + the same Pareto-margin nudge (treated as a target distribution shift via a KL-projection step), not by reward-weighted policy gradient.

5. **Vector feedback rule:** Channels enter the Pareto partial order on equal footing. There is no fixed weight `w`. The "step penalty" channel and the "precious" channels both contribute to coordinate-wise dominance; an action is considered better only when it dominates *across* the channels we care about. To avoid the FFTV/RG-style collapse where only one channel has non-degenerate signal, CID applies a **channel-active gate**: if `q_m` has training accuracy below a calibration threshold (e.g., AUC < 0.55 — meaning the head has not yet learned to discriminate firing from non-firing), channel `m` is excluded from the Pareto vote for that update. As more channels train up, the partial order recruits more dimensions — but the gate is reward-free (it depends only on the head's auto-supervised AUC, not on observed reward).

6. **Rollout-cost discipline:** One environment step per decision. No counterfactual branches, no simulator forks. Per update: O(k) supervised mini-batch updates on `q_m` + one supervised mini-batch update on `f_φ` + one policy logit-nudge step. Per accepted improvement: same, no acceptance test, no re-rollout. At deployment: one forward pass through `q_m` and `f_φ` per candidate action — `|A|` forward passes per decision — no extra env steps.

7. **Nearest-neighbor novelty audit:** The closest priors are CWAI (Jacobian column-norm of forward-model output per action per channel), JFP (Jacobian peak-time of forward-model cumulant prediction per action per channel), and the GVF/successor-feature disqualifier family. CWAI and JFP measure *gradients* of a learned next-observation/cumulant model — they collapse on stochastic transitions because Jacobian norms shrink toward the noise floor (this is exactly why CWAI passed DST but failed RG). CID measures the *log-probability shift* of a binary firing classifier between two stochastic next-state distributions — log-probability shifts of bounded probabilities do not shrink under added stochasticity (they re-weight, but the head is trained on the same noisy distribution, so the LR remains action-discriminative). GVFs/successor features predict *cumulative discounted firing magnitude*; CID predicts *one-step pointwise firing probability ratio*. The structural distinction: CID's primitive is a **one-step likelihood-ratio**, not a *cumulative magnitude* and not a *gradient-norm*. Pareto over likelihood-ratios is signed and information-theoretic; Pareto over GVF magnitudes would be magnitude-dominated and reduce to scalar success-rate maximization on the dominant channel.

8. **Predicted failure modes:**
   - **Terminal-only-channel head collapse.** If a precious channel fires only at terminal step, `q_m` may stay flat (predict 0 everywhere) until the first success enters the buffer; LR_m row is uninformative. Bootstrap risk equivalent to FED-family — but mitigated structurally by the channel-active gate which simply excludes the under-calibrated channel from the Pareto vote rather than letting it inject noise.
   - **Single-channel collapse to scalar.** If only the step-penalty channel passes the AUC gate, the Pareto vote degenerates to scalar maximization of `LR_step_penalty`, which is curiosity-rebadge territory (RND-like exploration suppression). This is the explicit falsifier above.
   - **Forward-model `f_φ` failure on stochastic transitions.** If `f_φ` cannot learn the next-observation distribution well (high entropy, e.g., random NPCs in RG), `LR_m` evaluated at `f_φ(o, a)` will be nearly action-invariant — the operator goes silent. Diagnostic: `f_φ` MSE per (o, a) bucket; if `f_φ` cannot distinguish across actions at most decision states, CID is structurally inapplicable.
   - **Policy-induced distribution shift on `q_m`.** As the policy improves, the channel-firing distribution on the buffer changes; `q_m` is chasing a moving target. If the rate of policy change exceeds `q_m`'s adaptation rate, `LR` is computed on a stale head. Mitigation: small `q_m` model + high learning rate; predicted to be a tuning issue, not a structural one.
   - **Action-invariant `LR` rows.** If `f_φ(o, a)` predicts nearly the same `ô` for all `a` (e.g., on early-training observations where the model has barely learned), all Pareto rows tie, and the nudge is zero. Predicted: this resolves once `f_φ` trains, but until then the operator is silent (acceptable — same as JFP's Jacobian-noise-floor failure mode but without the magnitude shrinkage problem).

9. **Side-information channel:** **vector diagnostics** (per-step `info["vector"]` supplies channel-firing labels for `q_m` supervision) + **learned dynamics** (`f_φ` is a one-step next-observation predictor trained on buffer transitions). No transition geometry beyond what `f_φ` learns; no event traces, no demonstrations, no language, no verifier feedback, no environment instrumentation beyond the panel-standard vector channel.

10. **Monotonic improvement claim:** Under stationary policy `π` and well-calibrated heads (`q_m` ≈ `P(channel m fires at next step | o)`, `f_φ(o, a)` ≈ `E[o_{t+1} | o, a]` in expectation), the Pareto-margin nudge moves probability mass from action `a'` to action `a` whenever `LR[a, :]` Pareto-dominates `LR[a', :]` — i.e., whenever taking `a` strictly raises the imminence probability of at least one channel and weakly raises imminence of all others. This is a monotone improvement in **the policy's induced expected one-step channel-imminence vector**, in the coordinate-wise partial order on ℝ^k. The claim is a Pareto-monotone improvement: no channel's imminence is reduced; at least one is increased. The claim does *not* extend to terminal cumulants directly — the monotone object is the per-step channel-imminence vector under the current policy. The bridge from "imminence increases" to "terminal cumulants increase" is not asserted; it is the empirical question the substrate run answers.

## Why it is not GVFs / successor features (or CWAI / JFP)

GVFs and successor features predict the *expected discounted cumulative firing* of channels — a magnitude — and policy improvement requires a downstream linear scalarization (as in successor-feature transfer with task-vector `w`). CID's primitive is a **one-step pointwise log-likelihood-ratio** between two binary firing probabilities, *not* a cumulative magnitude; the Pareto-non-dominance composition law means channels are never linearly combined or scalarized. CWAI and JFP compute *Jacobian-norm* and *Jacobian-peak-time* of a learned forward predictor — these are gradient-magnitude objects that shrink under stochastic env transitions (as CWAI's RG collapse demonstrates); CID's likelihood-ratio is a *probability-space* object that re-weights but does not shrink under the same stochasticity, because the binary classifier is trained on the same stochastic distribution it is later evaluated on. The structural distinction holds under variable renaming because the central object's *type* differs: gradient-norm vs. log-probability-ratio vs. cumulative-magnitude.

## Why it scales beyond the substrate

At horizon 20k actions with sparse terminal reward, value-function bootstrapping fails because the target `Q ≈ 0` everywhere. CID's primitive does not bootstrap from terminal reward at all — `q_m` is trained on per-step channel labels, which fire at intermediate steps for non-terminal channels (e.g., "generated-token-was-valid" or "tool-call-succeeded" in an LLM-with-tools setting) and provide dense supervised signal regardless of horizon. The channel-active gate ensures that channels which never fire are simply excluded from the Pareto vote rather than collapsing it. For LLM-with-tools, the action "generate a paragraph" can be scored without enumeration: `f_φ(o, a)` can be replaced by *a single rollout to the next user/env response* — at which point `LR_m` is computed *post-hoc* on the actually-observed `o_{t+1}`, with no enumeration over the action space. The policy update is then a soft KL nudge toward the action *type* corresponding to high LR Pareto-margin, computed from the ratio of post-rollout `q_m` shifts. For 6-component vector feedback (success, cost, safety, latency, validity, correctness), the Pareto partial order on ℝ^6 is exactly the structure CID's improvement operator already requires — no additional machinery, no fixed weighting, no preference model. The primitive's k generalizes from 2 (panel) to 6 (deployment) without changing the operator.
