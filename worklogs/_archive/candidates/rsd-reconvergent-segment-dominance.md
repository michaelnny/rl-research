RSD (Reconvergent Segment Dominance) maintains a directed multigraph over observation-hash nodes whose parallel edges store accumulated vector signals for all observed action segments sharing both a start observation-hash `u` and an end observation-hash `v`. The improvement operator finds each node pair `(u, v)` with two or more parallel edges, computes coordinate-wise Pareto dominance over the accumulated vectors, and behavior-clones the policy toward any Pareto-dominant edge with weight proportional to the count of segments it dominates — no scalar reward weight, no Bellman backup, no critic, no rollout verification. The structural distinction from SIT is real: SIT compares divergent suffixes from a shared prefix (downstream futures differ, requiring verification rollouts); RSD compares convergent segments sharing both endpoints, so the downstream distribution is identically shared by construction and no verification is needed. The run completed without crash but scored zero across all 4 envs (beat_random=0, beat_strong=0) because on sparse-reward envs without seeding, no reward-bearing closed witness pairs form and the operator is a no-op — exactly the bootstrap failure the hypothesis predicted. Evidence needed: an explicit random-exploration seeding phase that populates the multigraph with diverse trajectories before dominance updates begin, followed by verification that (a) Pareto-dominant edge pairs actually form, (b) the policy improves after updates, and (c) observation-hash collision rate is diagnostically low enough that dominance comparisons are trustworthy.
