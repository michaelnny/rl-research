CID (Channel-Imminence Differential) computes a per-(observation, action, channel) one-step log-likelihood-ratio `LR_m[o, a] = log q_m(ô_{t+1}(o, a)) − log q_m(o_t)` where `q_m` is a small supervised binary classifier trained to predict whether channel `m` will fire on the next step. The improvement operator is a Pareto-non-dominance vote on the k-dimensional LR vector, with an additive logit nudge proportional to the signed domination count `n_a − m_a`. This primitive is structurally distinct from CWAI/JFP (gradient-norm objects that shrink under stochastic transitions) and GVFs/successor features (cumulative magnitude + linear scalarization): the binary classifier's probability-ratio is trained on the same stochastic distribution it is evaluated on, so added stochasticity re-weights the LR rather than shrinking it toward a noise floor. The Reviewer passed this as `novel-direction`. The run failed because the implementation used a decision-time forward model `f_φ(o, a) → ô` to evaluate `LR_m` at each candidate action — if `f_φ` cannot distinguish actions early in training, LR rows are action-invariant and the Pareto nudge is silently zero throughout training (DST: 99.0 vs random 194.0; RG: 0.001 vs random 1.331). The hypothesis explicitly described an alternative "canonical version" that computes LR from actually-observed `o_{t+1}` post-hoc on replay buffer samples, with no forward-model query at decision time; this path avoids the bottleneck entirely and was not what ran. Next attempt should implement the offline canonical version: at each buffer update, for each transition `(o_t, a_t, o_{t+1}, v_{t+1})`, compute `LR_m = log q_m(o_{t+1}) − log q_m(o_t)` and accumulate per-(cluster, action, channel) running means for the Pareto vote — no `f_φ` at decision time.
